      某次面经

      Post on 2015-04-16

      上周四中午突然看到某街的招聘网页更新了校招的时间，居然是周五中午，权衡了下，当机立断买了车票赶往杭州。折腾了几天，终于拿到了的 offer。
      虽然笔试和面试是都是很简单的题目，应朋友的要求，写出来 share 一下。

      笔试：（共两小时）

      分为两部分，基础题是不管什么职位都要做的，然后是各个职位的题目。

      基础题：

      都是些基础的内容，我能想出来的就只有：

      选择题：

      1. 13 * 6 = 80 在什么进制下成立。
      2. 给入栈顺序，问合法出栈顺序。
      3. 设计模式的几个原则。
      4. 居然考 port channel，听都没听过，又不是考 CCIE。
      5. （后面的忘了。想起来再补充。）

      作答题:

      1. 有一组任务要完成，并且有些任务要在其它任务完成之后才能开始，每个任务都有自己的执行时间。求每个任务的完成顺序和完成时间。要求自己设计结构及算法。

        把每个任务看作图上的一下节点，就是红果果的拓扑排序，不是很难。
        不过如果用矩阵去存这个有向无环图会有点浪费，我是用每个节点维护一个 next 数组，以及当前节点入度。每次要取出入度为 0 的本来想用优先队列实现，感觉不靠谱（因为操作时会改变），最后就暴力了。

        事后的考虑：可能考虑并发和多线程会加分。

      2. 用户有不少优惠券，给你本次消费金额，求出最佳的那张优惠券。要求设计优惠券的结构及算法。

        遍历一遍，找出最优的。
        要考虑到的是：优惠券重复、折扣一样的情况。

      职位题（后台研发\应用开发\运维 都是同一套题）

      后台研发就两作答题。

      1. 俩有序数组，问它们有没有相同的元素。
        算法题。
        有几种做法：1. 暴力，时间复杂度O(n^2)；2. 二分，时间复杂度O(nlogn)；3. Hash，时间复杂度O(n)，不过要用O(n)的空间；4. 用双指针，O(n)时间，O(1)空间。
        很明显，第四种是最优的啦。
        还算不是很难，不过面试时提到了下，听说有很多坑，比如：数组有序不代表顺序一样，数组可能会非常大什么的。

      2. 进行性能瓶颈分析。
        这题应该是考运维的吧。不是很懂，就瞎扯了。（好像当时还扯蛮多的，面试时问我怎么想，我说脑补）
        具体可以看看这篇博文：性能测试瓶颈分析步骤。（我发现跟我脑补的差不多 = =）

      面试

      笔试完当晚就通知第二天早上面试了。
      第二天到公司，有不少人已经在等了，等了 40 分钟才轮到我，我也是体验到了一把面试前的等待了。

      一面

      第一面是技术面，首先自我介绍，然后问各种简历上的东西，各种挖简历上的东西，问项目遇到的问题，项目用到的技术呀。有些不是很清楚的只能各种瞎掰 TAT，幸好面试官也不是很懂，混过去了 =w=。（以后再也不敢在简历上乱写了）
      问了一项目里面 SQL 数据库的表，然后找了个点提了些建议。（事后看了一篇讲 MySQL 内部实现的文章，简直茅塞顿开，为什么没早点看呢）还问了个 grep 的简单使用。
      另外问我 Github 上的项目，我说就一些算法的东西和几个无聊做的 Win32 游戏和 Cocos2d-x 游戏，然后又是各种问游戏，我又不怎么玩游戏，没有聊起来，就聊了游戏的设计还有其中的算法。我说我只是玩玩，项目都很小，然后面试官就批评我不能小看自己的项目，我知道错了 >.<，我也没想到随便 push 上去的几个小游戏会被问到。
      然后聊笔试中的内容，我就讲了我后来发现的几个错误，就是有序数组那题里面的，他说那题坑很多，然后换了几个姿势来问我，问我大数据怎么办，我说大数据双指针也是可以的。
      然后要我用通俗的语言给女朋友描述栈，只能表示首先要有个女朋友 QAQ。我口舌那么笨，吞吞吐吐讲了下。然后又要我描述指针，感觉各种扣分 >A<。
      最后问我还有什么问题，我就问一下在这个职位能学到什么之类的问题（事先准备的问题，在这里 thx “侠姐”）。

      紧接着就把我带到楼上进行二面。

      二面

      现在想想，不知道二面算是技术面还是 HR 面。
      总之，面试官先告诉我，他这是最后一面了，也就是他是最终 BOSS。
      然后看简历问项目。（又是项目！）
      简单问了下简历上的东西后，就开始各种问家庭情况，为什么要选这个公司，有没有女朋友啦（sad story）。对了，还问我为何不考研，不出国。
      还问我有没有参加过其他的招聘，我表示这次是我的初面，所以会很紧江，有些回答很不清楚啊。他说你不要紧张，如果刚才面试回答不满意，现在可以补充一下，然后我就扯了下我的 ACM 队里的情况（我好机智）。
      然后就是例行问我有什么问题，我没准备太多这种问题，于是就问了几个现场想到的问题，问在这公司能变全栈么 >.<，发现好傻。
      然后就没有然后了，二面就这样结束了。

      面试就两面，说好的手写 KMP\Quick-sort\RB-Tree 呢。（开玩笑的）

      这就是我上周的面试经历，如果有想到新的内容再补充上去，如果有什么问题或建议请提出来。
      最后，非常感谢基友“侠姐”=w=。

      （PS，杭州空气的确不太行）
